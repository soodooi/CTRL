# Goal-Driven Development

`vault/ctrl/GOAL.md` is the single active goal owned by bao.

Before non-trivial work:

1. Read the file.
2. State which success criterion the task serves.
3. If no goal is set, ask bao; never invent one.
4. If the request does not serve the goal, identify the scope change before proceeding.

A goal must have verifiable success criteria and explicit non-goals. Mark it complete only after fresh evidence proves every criterion.
