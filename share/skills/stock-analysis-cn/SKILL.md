# A 股买卖决策分析 — 规范

## 铁律(每次都守)

1. **先复盘市场,再看个股 —— 单股结论必须带 regime 语境**:任何个股买卖判断前,**先定当日市场周期阶段**(`stock-cn_market_mood` → 冰点/修复/发酵/高潮/退潮),并让结论受该阶段**纪律**约束。脱离大盘情绪 + 板块强度孤立看单股 = 片面,禁止。同样的技术形态,在「退潮」和「发酵」里意义相反。阶段判定 + 每阶段纪律见 **`mood-cycle` skill**;完整日复盘(情绪+强度+归因)见 **`daily-review` skill** —— 用 `skill_read` 读它们,别自己另立一套。
2. **数据接地,绝不编**:每个结论必须基于**真拉到的数据**,并在该条后标数据源(如 `来源: stock-cn_stock_kline (yahoo)`)。拿不到就明说「拿不到」,绝不编造价格/指标/资金数字。
3. **定位 = 个人决策分析,不是荐股/投顾**:给分析和「买/卖/持有」判断供用户自己决策;不承诺收益、不自动下单、不替代持牌投顾。措辞用「分析显示 / 从数据看」,不用「推荐买入」式荐股口吻。
4. **先调工具再下结论**:先 `gate_tool_search` 看手上有哪些行情/股票工具,拉到数据后再分析。不空谈。

## 数据源 —— 首选 `ctrl-stock-cn` 功能包的 gate 工具(A股专用,免 key)

A股深度数据(情绪/强度/K线/筛选/报价)由 **`ctrl-stock-cn` 功能包**提供,工具以 `stock-cn_*` 挂在 gate 上。你(capped brain)默认列表里看不到它们 —— 用 **`gate_tool_search`**(如查 `"stock kline"` / `"market mood sentiment"` / `"screen strong"`)找到,再 **`gate_tool_call`**(`name=stock-cn_<tool>`, `args={...}`)调用:

- `stock-cn_market_mood` —— 大盘情绪:涨跌家数、涨停/跌停、炸板率、连板高度、昨涨停今溢价、**周期阶段**(ice_point/repair/ferment/climax/ebb)。盯盘/择时先看它。
- `stock-cn_limit_ladder` —— 连板天梯(强度):按连板数分组的领涨梯队。
- `stock-cn_sector_strength(kind, limit)` —— 板块强度榜(kind=`industry`|`concept`)+ 领涨股 + 主力净流入。
- `stock-cn_screen_strong(min_pct, min_volume_ratio, min_turnover, ...)` —— 全市场按 涨幅/量比/换手/成交额/流通市值 筛强势股(选股主力工具)。
- `stock-cn_stock_quote(symbols)` —— 逗号分隔代码的实时报价(价/涨跌/量比/换手/成交额/流通市值/今开高低)。
- `stock-cn_stock_kline(symbol, days)` —— 单只日 K 线(date/open/close/high/low/volume/pct),喂 MA/MACD/RSI。多源级联(Yahoo 全球可达 → akshare qfq → EastMoney 直连),返回里 `source` 标明用了哪个。

**没装包 / 工具搜不到**:说明 `ctrl-stock-cn` 未安装 —— 提示用户在 workbench 装它(或 `mcp_pack_install`),别退回凭记忆编数字。

**免费端点会抽风**:偶发 `ConnectionError`/超时(EastMoney/Sina/THS 免费源限流),工具自带重试+多源级联;某只票某工具报错时,如实说「这个源此刻拿不到」并换工具/稍后重试,**绝不编造数字补位**。基本面/消息用 `web_search` 补,但只作定性、不当精确数字源。

拉到 K 线后自算指标(不要口算):MA5/10/20/60、RSI(14)、量比、近 N 日高低。

## 四种任务

### ① 个股分析(regime-first:市场语境 → 技术 + 资金 + 题材 → 买/卖/持有)

> 顺序不能反:先定市场,再看板块,最后落个股。孤立的技术面读数不是分析。

0. **市场 regime(先做)**:`stock-cn_market_mood` → 周期阶段 + 该阶段纪律(见 `mood-cycle`)。这决定整体仓位倾向 —— 退潮期即便个股超跌也别博反弹(纪律优先于形态)。
1. **板块强度语境**:`stock-cn_sector_strength` 看该股所属板块强不强、在不在主线;逆势弱板块的个股,技术再好也降级。
2. **个股技术面**:拉报价 + 60~250 日 K 线 → MA5/10/20/60 排列、RSI(14)、量比、支撑/压力(近期高低 + 均线)。
3. **资金面**:主力净流入、北向、换手率(有数据源则拉,没有则明说缺)。
4. **题材/基本面**:所属题材热度、估值(PE/PB)历史分位、近期消息(web_search 定性)。
5. **结论**:买 / 卖 / 持有 + **止损位** + 目标位 1/2 + 盈亏比 R:R。**结论必须与阶段纪律一致**;若个股形态与市场纪律冲突(如退潮期的超跌反弹),明写「纪律优先,观望/不追」而不是硬给买点。每条标数据源。

### ② 选股(screen)
`market_screen` 或 EastMoney 全市场榜 → 按 涨幅/量能/主力资金/题材热度/技术形态 筛 → 输出 ranked 候选清单,**每只一句为什么**(信号 + 数据)。默认 5~15 只,不做长尾墙。

### ③ 止损/风控
对持仓&自选给**止损位 + 触发信号**。规则:跌破关键均线(MA20/MA60)、放量下跌、所属题材/市场情绪转弱、跌破前低。给「跌破 X 视为趋势转弱」这类可执行线。

### ④ 盯盘 / 复盘
盯盘 = 自选股实时异动 + 市场脉搏(赚钱效应、板块轮动主线/妖板/接棒、连板天梯)。
**用户说「复盘 / 今天总结」→ 走 `daily-review` skill 的三段式流程**(情绪面 + 强度面 + 自选归因 → 写 daily note + 追复盘表行),别在这里自己拼一套。

## 输出格式

- **决策先行**:开头一句给 买/卖/持有 + 止损位。
- **分点理由**,每条标数据源;能表格化就表格化(报价/技术/资金/题材各一块)。
- **止损 / 主要风险** 单列。
- **透明**:用户想看原始数据(K线/资金流)时,把拉到的原始值给他。
- 数据日期标清楚(A 股有交易日,别拿隔夜/停牌数据当实时)。

## 反模式(别做)

- ❌ **脱离大盘情绪 + 板块强度孤立看单股**(只给 MA/RSI 就下结论)—— 这是最常犯的「片面」,单股读数没有 regime 语境等于没分析。
- ❌ **违反当前周期阶段的纪律**(例:退潮期还建议「轻仓博反弹」)—— 纪律优先于个股形态。
- ❌ 没拉数据就凭记忆给点位/资金/业绩数字。
- ❌ 荐股口吻 / 承诺涨幅 / 自动下单。
- ❌ 把组合持仓记账(ghostfolio 那套)当成炒股分析 —— 用户要的是**买卖决策**,不是记账。
- ❌ 长尾罗列几十只票 —— 选股要收敛成少数高信号候选。
- ❌ 另写一套复盘/情绪判定 —— 复盘走 `daily-review`,情绪阶段+纪律走 `mood-cycle`,本 skill 只做个股深挖。
