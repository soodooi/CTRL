一个 manifest.json（纯数据） + 四层能力，装上就是一个 L1 入口。CTRL runtime 通用，加这个包写零行 kernel 代码。

00 功能包是什么

功能包不是一个 app，是一份声明 + 四层能力的组合。这个包的清单里全是数据——CTRL 读它、装它、渲染它，不为它写专门代码。作者字段写着 irisy · agent：它是 Irisy 按需求现写的，不是我手敲的。

# manifest.json —— 全是数据，无一行为它定制的 kernel 代码 "id": "ctrl-stock-cn", "name": "A-Share Assistant", "icon": "S", "author": { "name": "irisy", "kind": "agent" }, // Irisy 现写的 "variant": "mcp-server", "server": { "command": "uv run service/main.py" }, "ui_surface": "workspace", "workspace": { "table_prefix": "tables/stocks-" }, // ← 让它成为一个 L1 "knowledge_base": "projects/stock-cn", "skills": ["stock-analysis-cn", "mood-cycle", "daily-review"], "permissions": ["network"], // 免 key 公开源；无账号、无密钥、无下单

01 四层架构

这是最该看清的一张图。底下两层是能力（工具 + 方法论），Irisy 把它们用起来、产出落到智能表格，而每一次调用和写入都穿过 :17873 治理门。数据自下而上流。

④ 治理门 :17873 —— 每次工具调用 + 每次写操作都穿过它（审计留痕 · 写入过人工审批门）
③智能表格层
Irisy 的产出落进本机 vault 真表（tables/stocks-*）；你在工作区直接排序 / 筛选 / 色阶操作，像用飞书多维表。
stocks-watchlist 自选监控stocks-review 复盘stocks-screen 筛选结果
▲
Irisy（脑） —— 先读②方法论（regime-first / 结构优先），再按需调①工具，结论带失效点、不堆指标。
▲
②方法论层
3 个 skill（按需 vault_read），教 Irisy 怎么用工具——全从真实可用产品蒸馏，不是训练瞎编。
stock-analysis-cn 缠论结构优先mood-cycle 92科比四阶段daily-review 日复盘
▲
①数据 / 工具层
7 个 stock-cn_* 工具 = 一个本地跑的 mcp-server 服务（service/main.py），免 key、多源 failover。
market_moodlimit_laddersector_strengthscreen_strongstock_quotestock_klinestock_technical
为什么是四层而不是「一个工具」：裸工具只会返数字；方法论层让 Irisy 像个懂行的人那样先看市场 regime、再看结构、指标只当过滤器；智能表格层让结论可留存、可操作；治理门让每一步可审计、写入需你点头。这套「raw 工具 → AI-native 提升层」正是功能包的护城河。

02 三种用法

对话

主用法。Irisy 是 pipe：你说需求，它调工具、把结果路由进右侧工作区面板。

你：今天市场情绪怎么样？
→ market_mood → 周期阶段 + 每个数字附原文

直接操作

打开股票 L1 = 多 Tab 智能表格工作区。自选 / 复盘 / 筛选表，直接排序、筛选、色阶。

手动：在自选表拖列、改条件、看板块色阶热力——不经 Irisy 也能干。

定时任务

hermes 自带 cron。让它无人值守地按点跑。

定时：每天收盘后自动跑情绪总貌 + 复盘，写进当天笔记。
你对 Irisy 说它做什么
「今天市场情绪怎么样？」market_mood 情绪总貌 + 周期阶段判定（冰点 / 修复 / 发酵 / 高潮 / 退潮）
「连板天梯给我看」limit_ladder 最高板到 2 板全天梯，带板块归属
「筛量比>2、换手>5%、涨幅>5% 的票」screen_strong 多维筛选，结果落智能表格
「600519 的技术指标怎么样？」stock_technical 量比 / MACD / KDJ / RSI / BOLL（MyTT 公式，量价读法）
「把 600519 加进自选」写进自选智能表格（带实时行情列，过审批门）
「今天复盘」读 daily-review → 情绪面 + 强度面 + 自选归因 → 写进今天的 daily note

读法：3527 家上涨、涨停池 108 但炸板率 32.5%、最高只有 4 连板、昨日涨停股今天平均 −0.0%——情绪在发酵期：有赚钱效应但接力谨慎。每个结论都附原始数字，有据可查、不编数。

05 治理 · 主权 · 它怎么长大

  • 现写的源码就在 service/main.py（vim 能开、你能改）。想要新能力（比如龙虎榜筛选）直接跟 Irisy 说，它改代码加工具、重启即生效。
  • 数据主权数据本来就是你的，落本机 vault 的 markdown 表；免 key、无账号、拔网降级。
  • 治理门每次调用过 :17873 审计门；写操作（建表 / 写笔记）过人工审批门。
  • 不做没有任何下单能力——纯数据 + 管理工具，不是投资建议。
