ctrl-stock-cn · output ui reference

股票功能包该给用户什么 not JSON in a chat box

行业里成熟的炒股工具全在做同一件事:替用户读 —— 前面给一句结论(温度计 / 表盘 / 颜色 / chip),原始数字收进 drill-down。CTRL 现在把工具的 JSON 堆在聊天里,靠 Irisy 转述 —— 这就是「不友好」的根。下面每个展示面:左 = 别人怎么展示(带出处),右 = CTRL 该长啥样(mockup)。

色

承重规则:红涨绿跌 —— 和西方相反。大陆 红 = 涨/买入/资金流入,绿 = 跌/卖出/流出(港股才用西方绿涨)。绝不能继承西方图表库默认色。CTRL 的 --up / --down token 已经是对的,所有卡片沿用。来源 · BehavioralEconomics — When Red Means Go

01 大盘情绪 / 强度

当前:market_mood → {"up_count":3527,...} 一坨 JSON。目标:一眼冷热 + 一句读法。

别人怎么展示

情绪温度计 0–100° —— 同花顺「市场情绪温度」原生范式。>80° 过热(卖区)、<20° 冰点(买区),买冷卖热直接刻进刻度。比西方表盘更贴中文直觉。同花顺 情绪指数 883404 · 10jqka

涨跌家数比例条 + 大数字块 —— EastMoney 把全市场拆成「涨停股池 / 炸板股池 / 强势股池」可导航池;一条涨/跌比例条 + 涨停/炸板率几个大数字,不用图就知道今天健不健康。EastMoney 涨停板行情

半圆表盘 + 一词判定 —— CNN Fear & Greed:色弧 + 指针 + 中心大数字 + 一个词。零指标素养可读。CNN Fear & Greed

02 龙头 / 龙虎榜(聪明钱)

排名 + 一个核心指标(净买额)+ 谁在买。数字别裸奔。

别人怎么展示

格子内嵌条(bar-in-cell) —— 每行主指标(净买额/涨幅)在单元格内画一条横条:长度=大小、颜色=正负。扫条长,不读数字。TradingView heatmaps · Finviz top gainers

带符号的颜色净额列 —— EastMoney 龙虎榜:净买正 红、净卖 绿。颜色本身就是信息:红行 = 钱在流入。EastMoney 龙虎榜

03 连板天梯(A股独有)

按连板高度分层,最高板钉在顶 —— 位置 = 强度排名。

别人怎么展示

竖向分层天梯,最高板置顶 —— 同花顺「连板天梯」:龙头/最高板在顶,逐层降到首板,「1板2板3板排得清清楚楚」,角上带「上一日/下一日」翻页。竖直位置本身就是强度序,零指标知识。雪球 使用指南

股池 tab 作顶层切换 —— 涨停 / 连板 / 炸板 分名字桶,按意图点 tab,不用筛。带汇总行「连板总数9只,三连板以上4只」。EastMoney ztb

04 个股技术面

MA/MACD/KDJ/RSI/BOLL 一堆指标 → 给外行一个判定,不是一张公式表。

别人怎么展示

表盘 + 5 档判定弧(强卖→强买) —— TradingView Technicals:半圆红→灰→绿,指针指向汇总,中心一个词。并列三表盘(振荡/汇总/均线)。TradingView gauges

买/中性/卖 计票 —— 表盘下三个计数「12 买 · 6 中性 · 8 卖」喂给指针:既是判定,又给了理由,不用盲信。Webull charts

多维诊断卡 + 百分位分「打败58%」 —— 同花顺牛叉诊股:技术/资金/消息/行业/基本五张面-卡各带分 + 综合评分;妙想把判定说成具体支撑/压力位。百分位把抽象分变成能感知的排名。同花顺 牛叉诊股

05 AI 解读(Irisy 回答)

结论先行 + 图内嵌 + 追问 chip + 一键看原始。这是「JSON 堆聊天」的直接解药。

别人怎么展示

人话提问 → 自动配列的可排序表 —— 问财 iWenCai 招牌:问题解析成筛选条件,结果列跟着问题生成(问估值出 PE,问技术出 MACD),渲染成可排序表。用户不学语法、不选列。同花顺 问财

06 五条贯穿原则

结论先行(BLUF)每个回答第一行是判定(偏多/退潮),证据卡按重要性递减排下面。倒金字塔。
AI 回答要可扫,不是散文直答 + 卡片/表/bullet + 留白 > 段落。给追问 chip。这就是「JSON 转述」的正解。
颜色翻转:红涨绿跌红=涨/买/流入,绿=跌/卖/流出(港股除外)。所有卡片用 --up/--down。
渐进披露 / 一键看原始默认给总结卡,长按展开原始数字(~2 层)—— CTRL 早就主张的「三层视图」。
常驻看板 > 每次都问永真状态(情绪温度 / 自选 / 龙头)推到常驻可扫面板,用颜色/大小/位置的前注意属性;聊天只留给一次性 NL 提问 —— 别让用户「问」本该一直看得见的东西。

07 落地:最小 → 终态

当前 = 聊天机器人 + 一张表 · 目标 = 炒股工作台(卡片替用户读)
A
结构化输出渲染(先做,最止痛) — Irisy 对 stock 工具的 JSON 统一渲染成上面这些卡片 + 一句读法。复用现有 smart-table viewer + 一个轻卡片组件。同时解决「输出太生 + 落点分散」。
验证优先卡:情绪温度计 · 龙头 bar-in-cell · 天梯 · 技术表盘 · AI 总结卡。
B
起手 chips — 开包即见「今天龙头 / 大盘情绪 / 今天复盘」,一点 → Irisy 跑 → 落工作区。复用 intro 里已有的例句表。
解决「进门不知问啥」。更小,可与 A 并行。
C
今日驾驶舱(终态,先不做) — 开包第一屏 = 常驻可扫面板(情绪温度 + 龙头表 + 天梯)。大工程,等 A/B 验证后再上。
符合「常驻看板 > 每次都问」,但按极简原则押后。
