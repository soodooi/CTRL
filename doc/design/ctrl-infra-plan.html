04 用户分享 HTML / 产物

你说的「用户能直接分享 html 或其他」——机制就是 R2 + Worker，跟我给你发 artifact 链接一个道理，且 R2 出口免费 = 分享再多不烧钱。

1本机产出（报告 / 面板 / 包介绍 = HTML 或 markdown）—— 数据始终在你 vault。
2点「分享」→ 上传到 R2，Worker 发一个短链。
3私有默认：链接带不可猜的 token；要更严就加口令 / 有效期 / 随时撤销。
4对方打开链接即看，无需装 CTRL、无需账号——符合「无账号系统」。

05 反馈收集：薄且尊重主权

opt-in、发送前用户审阅；只收够做决策的信息，不做监控档案。CTRL 独有的一招 —— 附上 gate 审计轨迹（event-store.db 的真实工具调用），你收到的是「实际发生了什么」而非用户转述。

崩溃单独路：opt-in panic-hook → 写本地 crash 文件 → 下次启动提示「要发送吗」，绝不静默上报。截图走 Tauri 抓 webview，敏感内容审阅门里可删。
