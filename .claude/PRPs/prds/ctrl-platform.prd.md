# CTRL — 一人公司的 AI 工作平台

> **品牌**：CTRL · **公司**：CTRL App Lab · **域名**：ctrlapplab.com
> **定位一句话**：按 `Ctrl` 唤出 → 说出 intent → AI 路由 → 调动「动作 / 专家 AI / 真人专家」三层供给完成工作。
> **本质**：双供给侧平台（不是工具）。CTRL App Lab 不做内容，**做让 OPC 互相服务的基础设施——而我们自己就是首批 OPC 创作者**。

---

## Problem Statement

一人公司主理人（OPC, One-Person Company）独自承担开发、营销、客服、财务、设计等全部职能，但现有 AI 工具碎片化（ChatGPT / Cursor / Granola 各一个），既没有共享生态、也没有把"问题→执行"做成秒级桌面入口。同行间的最佳实践只能截图发微信群，无法工程化复用。结果：OPC 重复造轮子、单兵作战、专业领域问题（法律/财税/设计）只能高价找咨询或赌运气问 AI。**不解决的代价**：在"AI 一人公司"这个 18-24 个月的黄金窗口期，没有专属基础设施的 OPC 群体会被切碎到各种通用 SaaS 里，CTRL 错过赛道定义权。

## Proposed Solution

CTRL 是 **Win + Mac 双端桌面平台**，按 `Ctrl` 单击唤出全局输入框，自动捕获场景上下文（选中文本 / 当前应用 / 剪贴板 / 屏幕区域）。用户输入 intent，AI 路由器决策走三层供给之一：
1. **L1 动作 / Playbook**——创作者发布的可执行步骤链（"把这段英文改成知乎风格"）
2. **L2 专家 AI persona**——领域专家把自己蒸馏成 AI（"问问张律师 AI 这条款合不合规"）
3. **L3 真人专家接管**——AI 答不了 / 用户付费要深度时一键转人工（"约张律师 30 分钟"）

CTRL App Lab 不生产任何 L1/L2/L3 的运营内容——我们做规则、SDK、AI 路由器、支付、争议、声誉系统。**唯一例外**：作为 founding creator，团队会发布 30-50 个种子 L1 动作 + 1 个创始人 L2 专家 AI + 6 个月内开放 L3 真人付费——这是冷启动的"播种"，不是长期运营。**为什么这个组合赢**：Quicker 只有 L1（无 Mac、无 AI）；Raycast 只有部分 L1+L2（订阅墙、不开放分享）；Delphi 只有 L2（无 L1、无 L3、无桌面入口）；知识星球只有 L3（无 AI、无桌面）。**没有人做过 L1+L2+L3 + 桌面 intent 入口 + OPC 圈层 + Founder-as-Creator dogfooding 路径。**

## Key Hypothesis

**我们相信**「桌面 `Ctrl` intent 入口 + L1+L2+L3 三层供给市场 + 创始团队亲自做种子创作者」会让一人公司主理人 **将日常重复任务的解决时间从分钟级降到秒级，并把同行最佳实践从"截图分享"升级为"可订阅、可执行、可付费"的资产**。
**我们将在 16 周内验证这一假设**：≥ 30 位种子 OPC（含创始团队）既是用户也是创作者；≥ 100 个公开动作 + ≥ 10 个上线的专家 AI；≥ 5 笔 L3 真人付费成交；首批用户 7 日留存 ≥ 35%。**如果做不到，平台模型不成立，转单边工具或退出。**

## Users & Context

### Primary User：OPC（一人公司主理人）

- **画像**：1 人创业者 / 独立开发者 / 独立内容创作者 / 自由设计师 / 自由咨询。中文圈，25-45 岁，PC 是主战场（每天 6+ 小时桌面操作）。年付费意愿 ¥500-3,000（已为 ChatGPT Plus / Cursor / Notion / Claude API 付费的群体）。
- **当前行为**：用 5-10 个 AI 工具切换工作；遇到跨域问题（法律、财税、设计、SEO）要么硬问 ChatGPT 要么花高价咨询；自己摸索的最佳实践只能写在 Notion / 微信群里。
- **触发场景**：
  - "我刚收到投资人的 term sheet，哪条要重点谈？"——L2 投资人 AI / L3 真人
  - "把这段英文改成知乎风格"——L1 动作
  - "我这个 React 报错怎么修"——内置或 L1 开发动作
  - "我老板让我签竞业协议要不要签"——L2 律师 AI / L3 真人
  - "帮我把今天会议录音转纪要 + 待办清单同步到飞书"——L1 多步动作
- **成功状态**：80% 重复任务在 3 秒内完成；专业问题在 30 秒内拿到比"硬问 GPT"质量高 10 倍的答案；自己的本职专业能被蒸馏成可订阅资产。

### Job to Be Done

> **当我**作为一人公司面对一个跨域专业问题或重复任务时，
> **我想要**一秒唤出一个理解我场景的 AI 入口，让它路由到最合适的能力（无论是动作、专家 AI、还是真人），
> **以便我**不用切应用、不用想找谁、也不用反复 prompt 工程，就能拿到比通用 AI 高一个量级的结果——并能反过来把我自己的专业蒸馏成被别人订阅的资产。

### Non-Users（明确不服务）

- **大公司员工**：有团队、有专业部门、有 IT 限制；CTRL 的 OPC 圈层心智会被稀释。
- **纯娱乐用户**：去 Character.AI / 筑梦岛。
- **手机原生用户**：桌面入口对你无效。
- **完全不写代码 / 不做内容 / 不接客户的人**：你不需要平台，需要的是单一 SaaS。

### MVP Scope（16 周，1 人，Win-only）

⚠️ **范围警告**：用户原始预期 4-8 周。1 人完成所有 Must 项的现实估算 = **14-18 周**（Tauri 学习 2 周 + 全局热键/上下文 3 周 + AI 运行时 3 周 + 创作者编辑器 4 周 + 市场+账号+支付 4 周 + 联调/打磨 2 周）。**取 16 周作为承诺**，砍版方案：先砍 L2/L3 进 V1.1，MVP 只做 L1 闭环 → 8-10 周可完成。

**MVP 必含（Path A：完整 16 周）**：
- 单击 Ctrl 唤出 + intent 输入 + 场景捕获
- AI 路由器（MVP 简化为：所有 intent 先走 LLM 总结 → 推荐 3 个 L1 动作）
- L1 运行时：LLM 调用 / 变量 / 流式输出
- 创作者发布工具：可视化拖拽 + 内嵌 JS / Python 脚本步骤
- 动作市场：列表 / 详情 / 一键安装 / 版本更新
- 账号 + Stripe + 微信支付（仅订阅；L3 真人付费 V1.1）
- Win 端打包 + 自动更新
- **团队 founding creator 自用**：每天用 CTRL 干活 + 边用边发动作

**MVP 砍版（Path B：8-10 周，推荐 1 人独立验证）**：
- 上述 1-7 全保留
- 砍：创作者发布工具的脚本步骤（仅可视化）；砍：支付（全免费）
- 价值：先验证"OPC 愿不愿意按 Ctrl 用 + 愿不愿意上传动作"，再投精力做商业化

### User Flow（最短价值路径）

**用户侧（消费者闭环）**：
```
按 Ctrl → 输入框唤出 + 场景自动注入 → 输入"把这段改成知乎风格"
  → AI 路由器：识别为 L1（写作改写）+ 场景含选中文本
  → 推荐 3 个动作（"知乎风格改写 by @作者A"等）+ 默认置顶最匹配
  → 一键运行 → 流式结果回写到原选中位置 / 复制到剪贴板
  → 满意：点赞 + 关注作者；不满意：转 L2/L3
```

**创作者侧（供给闭环）**：
```
登录 → 我的工作台 → 新建动作 → 命名 + 描述 + intent 关键词
  → 拖拽步骤（输入 → LLM 调用 → 输出 → 写回）→ 测试
  → 发布到市场（设置免费/付费）→ 收入仪表盘看安装数 + 收入
```

**团队 founding creator 闭环（独有）**：
```
团队某成员日常工作中遇到重复任务（如"把推文改成 LinkedIn 长文"）
  → 在 CTRL 里现编一个动作解决自己 → 跑 3-5 次确认稳定
  → 一键发布到市场 + 写一段开发笔记发 X / 即刻
  → 引来同类 OPC 装机 + 模仿创作 → 闭环自我加强
```

## Founder as Founding Creator（创始团队即创始创作者）

> **核心定位**：CTRL App Lab 团队（含创始人）本身就是 OPC，亲身经历用户痛点。**这不是一句口号——而是冷启动策略的核心、产品质量的兜底、营销叙事的母题。**

### 五重身份矩阵

| 角色 | 团队具体做什么 | 战略价值 |
|---|---|---|
| **Founding User** | Day 1 起每天用 CTRL 完成所有 OPC 工作（产品/工程/营销/客服/财务） | 真实使用 = 真实 bug + 真实 feature 优先级；不会做错方向 |
| **Founding Creator (L1)** | 团队发布前 30-50 个种子动作（开发常用、营销常用、文案常用、自动化常用） | 解决"空货架"冷启动；每个动作即"标准模板"供其他创作者参考 |
| **Founding Creator (L2)** | 创始人把"产品 / 技术 / 工程 OPC 经验"蒸馏成第一个专家 AI | 演示形态 + 引来同类 OPC 入驻；让"OPC 蒸馏自己"这件事有先例可循 |
| **Founding Creator (L3)** | 创始人在前 6 个月开放真人付费咨询（独立产品咨询、技术咨询） | 验证 L3 闭环 + 拿真实 take rate 数据 + 收集第一批付费用户访谈 |
| **Builder-in-Public** | 每个动作发布、每次 L3 接单都公开发 X / 即刻 / 播客 | 内容资产 = 双倍价值（自用 + 上架 + 营销）；运营成本 ≈ 0 |

### 为什么这是制胜策略（非可选）

- **"空货架"是平台死亡的第一原因**——创始团队的 30-50 个高质量动作让 Day 1 的用户体验从"空的"变成"立刻能用"
- **"OPC 双重身份"必须有人示范**——光说"你们也可以发布"没人信；创始人亲自既消费又创作，把路径走通
- **L2/L3 入驻最大门槛是"我去做这个值得吗"**——创始人率先把自己挂上去做出第一笔成交，是最强招募书
- **运营成本对比**：
  - 传统平台：招团队写动作 → ¥30-100/动作 × 50 = ¥1,500-5,000；雇 BD 招专家 → 月成本 ¥10,000+
  - CTRL：团队工作就是产物 → ¥0；创始人自己接 L3 单 → 反而是收入
- **品牌叙事**：每篇 builder-in-public 帖子都是"看，CTRL 怎么解决我自己的 OPC 痛点的" → 真实、不可伪造、引发同感

### 退场条件（必须设）

⚠️ Founding Creator 不能长期占据生态头部位置——会变成 *Quicker 头部作者断更后用户没替代* 的镜像问题。

**退居二线触发条件**（任一达成）：
- 平台总创作者 ≥ 200
- 平台总动作 ≥ 1,500
- 创始人 L2 / L3 收入占平台总 GMV ≤ 5%
- 内测期满 6 个月

**退居方式**：
- 团队动作进入"维护模式"（不再新增），但保留更新
- 创始人 L2 / L3 价格上调 5x（让位给中长尾专家）

## Technical Approach

**Feasibility**: **HIGH**——所有关键能力都有成熟开源/SDK 支撑，技术风险可控。

### 架构选型

| 层 | 技术 | 理由 |
|---|---|---|
| 桌面壳 | **Tauri 2.0** | <0.5s 启动、~30MB 内存、原生热键、跨端、Rust 内核 |
| UI | React + TypeScript + Vite | 创作者扩展用同栈，降低 SDK 学习成本 |
| 全局热键 | tauri-plugin-global-shortcut + 平台原生事件钩子 | 单击 Ctrl 检测需用底层 keyboard hook（Win: SetWindowsHookEx; Mac: Quartz Event Tap） |
| 场景捕获 | 平台 API：UI Automation (Win) / AX API (Mac) | 选中文本 / 当前应用 / 剪贴板 |
| AI 运行时 | Anthropic SDK + OpenAI SDK + DeepSeek + Ollama | 多供应商，用户可选 / 创作者可指定 |
| 路由器 | LLM (Haiku/Sonnet) + 规则引擎 | MVP 简化版：embedding 召回 + LLM 决策 |
| 创作者编辑器 | React Flow + Monaco（脚本步骤） | 拖拽节点图业界标准 |
| 后端 | Bun + Hono + PostgreSQL（动作元数据 + 用户）+ S3（动作产物） | 1 人友好；Bun 性能足够 |
| 支付 | Stripe（国际） + 微信支付 / 支付宝（中文） | 国内必含 |
| 自动更新 | Tauri Updater | 内置 |

### 关键技术风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 单 Ctrl 唤出与 IME / 修饰键冲突 | **高** | 误唤出 / 漏唤出体验崩 | 必须上线前做 100+ 用户实测；提供"双击 Ctrl"作为可选；冲突应用白名单 |
| 场景捕获 API 在新版 Win/Mac 受限 | 中 | 上下文质量下降 | 关键能力 fallback 到剪贴板；申请系统权限引导 |
| AI 路由准确率 | **高** | 用户拿不到对的动作就流失 | MVP 阶段允许用户手动 override；记录路由日志持续 fine-tune |
| 创作者动作恶意代码（脚本步骤） | 中 | 平台信任崩塌 | 沙箱化（Deno 子进程 / WASM）+ 上架前自动扫描 + 用户安装前明示权限 |
| L2 蒸馏质量绑定专家声誉 | 中 | 头部专家不敢入驻 | 创作者发布前必看 AI 输出抽样 + 用户协议明示"AI 言论不代表本人" |
| 中文支付 + 抽成的合规 | 中 | 卡在收款 / 提现 | 优先 Stripe Atlas 起家；微信支付走个人开发者商户 |
| Tauri 2.0 在 Win 端某些边缘 API 不稳定 | 低 | 工程返工 | 提前 spike（见 Phase 1） |

### Phase Details

**Phase 1: Spike（Week 1）**
- **Goal**：验证最大技术风险——单 Ctrl 唤出在 Win 上能否做到无误触
- **Scope**：Tauri 2.0 项目脚手架；全局键盘钩子识别"Ctrl 按下→释放且期间无其他键"；弹窗显示当前选中文本
- **Success signal**：连续 1 小时正常打字（含 Ctrl+C/V/Z/A）零误触；从按 Ctrl 到弹窗 < 200ms

**Phase 2: 用户侧 MVP（Week 2-5，与 Phase 3 并行）**
- **Goal**：跑通 "intent → 路由 → L1 → 结果"
- **Scope**：输入框 UI；调 LLM 做 intent 分类 + 推荐动作；L1 运行时（LLM/变量/流式）；写死 5 个内置动作（改写/翻译/总结/SEO 标题/邮件回复）
- **Success signal**：5 个动作全可正常运行；端到端 < 3 秒首字

**Phase 3: 创作者侧 MVP（Week 2-5，与 Phase 2 并行）**
- **Goal**：让创作者能在不写代码的情况下发布动作
- **Scope**：React Flow 拖拽编辑器；动作元数据（名/描述/intent 关键词/封面）；保存到本地 + 上传到后端；市场列表 + 详情 + 一键安装
- **Success signal**：自己测试发布 3 个动作；另一台机器能搜到、装上、跑通

**Phase 4: 账号 + 后端（Week 4-6，与 2/3 部分并行）**
- **Goal**：从单机走向云
- **Scope**：Bun + Hono；Postgres schema（users / actions / installs）；JWT 鉴权；动作上传 API + 下载 API；S3 兼容存储（用 R2）
- **Success signal**：注册/登录/发布/安装全链路通

**Phase 5: 闭环联调 + 内测（Week 7-10）**
- **Goal**：找到第一批种子 OPC 验证 PMF
- **Scope**：打磨流程；接埋点（PostHog）；招 30 位种子用户（V2EX/即刻/X 中文/播客社群）；每周 1v1 访谈 5 位；**团队同步发布 30-50 种子动作 + 创始人 L2 上架**
- **Success signal**：≥ 30 装机；≥ 100 动作（含团队 30-50）；7 日留存 ≥ 35%；至少 5 位社区用户主动发布动作（不含团队）

**Phase 6: 商业化 V1（Week 11-13）**
- **Goal**：跑通付费链路
- **Scope**：Stripe + 微信支付（个人开发者商户）；订阅 Pro（取消 LLM 调用配额）；动作付费（创作者定价 + 平台抽成）
- **Success signal**：≥ 5 笔真实付费；首月 GMV ≥ ¥5,000

**Phase 7: L2/L3 Beta（Week 14-16）**
- **Goal**：验证双供给侧第二、第三层
- **Scope**：L2 蒸馏（先集成 Delphi API 或 HeyGen，不自研）；L3 接 Cal.com + Stripe 一次性付款；招 5 位种子专家入驻；**创始人 L3 真人咨询同步开放**
- **Success signal**：≥ 5 个 L2 上线（含 1 创始人）；≥ 1 笔 L3 真人付费（创始人或他人均可）

**Phase 8: macOS 移植（Week 17+）**
- **Goal**：扩到 Mac 用户群
- **Scope**：Tauri 已抹平 ~80%；重写：键盘钩子（Quartz）、AX API、AppleScript 桥
- **Success signal**：Mac 端达到 Phase 5 同等闭环

### 网络效应模型

- **L1 单边效应**：动作越多 → 用户越多 → 创作者收入越高 → 更多动作

- **关键 metric**：创作者-用户重合率 ≥ 40% 是 CTRL 健康的灵魂指标

## Research Summary

### Market Context（来自两轮调研）

- **Quicker 模型够强但只在 Win 且无 AI**——5 亿动作装机 + 8000 公开动作证明需求；Mac 缺位 + 无 AI 是缺口
- **Raycast Pro AI 是订阅墙且不开放分享 workflow**——验证桌面 AI 入口商业化可行，但留出"开放共享 workflow"的真空
- **Delphi.ai $16M A 轮（Sequoia/Anthropic）**——蒸馏 15 分钟可成；纯 Web 无桌面无人工兜底是缺口
- **Character.AI / Talkie / Pi**——personal AI 单边产品天花板已现，"创作者赋能"路线被验证
- **知识星球（洪灏 880 万半月）+ 得到 IPO 6 次失败**——中文知识付费向"双向、个性化、即时"转型的窗口
- **Topmate 30 万创作者 Top 5% 月入 420 元**——纯人工时间售卖不可持续，AI 杠杆是必需
- **Cameo 75/25 分账**——头部入驻临界点，CTRL 抽成不能高于 25%

### Technical Context

- **Tauri 2.0 已稳定 Win/Mac/Linux**，全局热键、托盘、系统集成插件齐备
- **单 Ctrl 唤出技术可行**（Win 用 SetWindowsHookEx WH_KEYBOARD_LL；Mac 用 Quartz Event Tap），但必须实测兜底
- **AI SDK 全部成熟**：Anthropic / OpenAI / DeepSeek / Ollama 都有稳定 SDK 与流式 API
- **L2 不需要自研**：Delphi API + HeyGen + 11Labs 可直接拼装
- **L3 不需要自研**：Cal.com + Stripe 一次拼好

### 关键不确定性（PRD 之后必须做的事）

1. 30 位种子 OPC 实地访谈：验证"OPC 愿意按 Ctrl + 愿意发布动作 + 愿意做 L2 专家"
2. 单 Ctrl 唤出 100+ 用户实测：误触率与体验感
3. AI 路由器准确率：MVP 上线后 2 周数据
4. L2 蒸馏成本：自研 vs Delphi API 价格对比
5. 中文支付合规：Stripe Atlas vs 个人开发者商户的路径选择
6. Founder L2 / L3 上线后的真实数据：作为整个平台 take rate 与 user willingness to pay 的早期信号
