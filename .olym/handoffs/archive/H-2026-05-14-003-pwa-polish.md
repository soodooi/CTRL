## 你的 sub-task 列表 (按价值排, 不强制顺序)

### Tier 1 — 用户立即感知的差异化 (优先做)

1. **KeycapCard 视觉精化** — 真正像物理 keycap (3D 凸起 / press 时 inset shadow / 释放回弹 spring, OP-1 风格). 当前是个有 border 的卡片, 太弱.
2. **Pool 网格交互** — hover spring, keyboard 方向键导航 (Pool 是 launcher, 键盘交互比鼠标重要), 数字键 1-9 + 0 选前 10 个 keycap (Quicker 路数)
3. **Workspace 真实 Cell 流渲染** — Tauri WS bridge 已经 wire, zeus 后端的 stss_bridge 推 Cell. 你接 ws://, 订阅 stream_id, 实时渲染 cell list (类似 chat stream). Cell.kind 不同要不同视觉 (UserInput, LlmResponse, McpToolResult, ContextSnapshot 各自的 chrome).
4. **ClockStrip 视觉重做** — 现在是大字 + 小字, 应该有 OP-1 calc-style 7-segment / mono LED 感, 加 weekday 横排 progress
5. **Logo 应用到 Pool 顶部 + Settings + Workspace title bar** — `<svg use href="/icons/logo-mark.svg">`. 自家 logo 现在零曝光.

### Tier 2 — 产品完整度

6. **Settings 真东西**:
   - Marketplace 区: 列 MCP servers (zeus 后端 `list_mcp_servers` command), 每个 1-click install (zeus 后端 `install_keycap_from_mcp`)
   - BYOK 区: 表单输入 Anthropic / OpenAI key, 保存到 keychain (`store_key`), 显示当前已配置 (`get_key`)
   - **Hotkey config**: bao 2026-05-14 明确要求 default lone-Ctrl, settings 可配. 加一个 "录制快捷键" 控件 (用户按一组键, 显示 + 保存). 当前后端不接受 — 你出 UI + 暂存到 localStorage, 等 zeus P6 加 `set_hotkey` command 后接.
7. **Pool empty state** — 全新装机时 listKeycaps 返回 [] (sub-PR d 加了 5 个 seed 所以不会空, 但 mock data 替换后会). Empty state 需要引导用户去 marketplace, 视觉上是 "你的 keycap 池等待第一个工具" 这种诱惑.
8. **Workspace close 按钮 / Esc 关闭** — 当前 workspace 窗口只能从 taskbar / Quit 关. 加显式 close 按钮 (顶部 right) 调 zeus 后端 `close_workspace` command (zeus 加这个 command, 你 wire UI).

### Tier 3 — 细节 + 边界场景

9. **Vite/kernel 失联 friendly fallback** — bao 之前看到 "无法访问此页面" 是 Chromium 原生错误页. 加一个 service-worker fallback page: 检测到 `ws://127.0.0.1:17872` 连不上时, 显示 "CTRL kernel offline" + retry button.
10. **PWA install prompt** — mobile 用户加主屏体验 (manifest.webmanifest 已配, 加 `beforeinstallprompt` 事件处理 + custom UI 引导)
11. **Reduced motion / a11y** — `prefers-reduced-motion: reduce` 时所有 Framer Motion / CSS transition 降级
12. **i18n 框架** — 当前 hardcoded 中英混杂, 加 `react-i18next` 或类似. zh-CN 默认, en 备用. tokens.md 里 hint copy 是中文, 应该是 i18n key.
13. **视觉 token 应用审计** — 跑一次 `grep -rE "#[0-9a-fA-F]{3,6}|rgb\(" packages/ctrl-web/src/` 找硬编码颜色, 替换为 `var(--ctrl-blue)` 类 token 引用. brand-tokens.md 规则: 永远不硬编码颜色

### Tier 4 — 后期 (与 P5/P6 同期)

14. **AI 创作向导 UI** — slot-filling chat 风格, P6 阶段开. 不在你这一个 handoff 范围, 但你帮助设计 IA + 占位 route.
15. **Multi-window orchestration** — bao 后续会有 sub-PR 让 1 keycap = 1 独立 workspace 实例 (不是单 workspace 复用). 你帮 zeus 设计 multi-window UX (alt-tab 切换 / 多个 workspace 视觉区分).
