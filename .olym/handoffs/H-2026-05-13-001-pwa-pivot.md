## 现象

ADR-001 锁定的"L0 Tauri + native UI per-platform (WinUI 3 / SwiftUI)"路径在 W3.1–W3.6 全栈 Win 实现后, 暴露三个产品现实 (详见 ADR-002 §2):

1. 跨设备"share URL → 手机看流 → 反向发 Op"是定义性 UX (bao 2026-05-13 09:55 prompt)
2. 双 native UI 栈 (WinUI 3 + 未来 SwiftUI) 翻倍 UI 工作量、撕裂设计系统、拖慢视觉迭代
3. Web stack 迭代速度 (Vite HMR / 设计 token / 组件库) 远超 native

bao 2026-05-13 拍板: **PWA 方向重构**, 选 P2 (PWA + 极简 native shell), Tauri 2 作为壳 (zeus 推荐, bao 默认接受).

本 handoff = ADR-002 的执行级落地, 把 W3 的 UI 层换成共享 PWA, 保留 W3 的 hotkey/tray/MCP 验证成果。

### 风险

1. **Tauri 2 plugin 不稳**: global-shortcut + tray plugin 是 2.x 早期, 偶发 bug。Mitigation: 在 Step 3 做 spike, 验证 7 天稳定再继续 Step 4
2. **WebView2 在老 Win 缺失**: Tauri 2 bundler 可以打 bootstrapper, 加 ~20MB 安装包。Mitigation: 双 channel (online installer / offline bundler), 文档说明
3. **Service worker bug 锁住用户**: PWA 老 cache 不更新。Mitigation: skipWaiting + cache versioning + 服务器端 kill-switch header
4. **mobile WS 后台断开**: iOS 后台立即断 WS。Mitigation: 接受这个事实, 用 push 唤起后端推送
5. **CSS 反 template 难度**: 默认 React 项目长得像 ChatGPT 灰。Mitigation: Step 5 第一周做视觉 spike, 先定 token + 一个标杆组件 (KeycapCard), 否则后期改成本高
6. **kernel daemon 启动赛跑**: shell 起来时 daemon 还没 ready, hotkey 收到事件但 invoke 失败。Mitigation: shell 在 daemon 报告 ready 之前不注册 hotkey
