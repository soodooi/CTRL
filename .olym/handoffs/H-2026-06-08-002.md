## 现象

macOS 是主线、Windows 是副线 (bao 2026-06-08)。Irisy 在 mac 上领先,Windows 上有
平台破绑。目标:在 Windows 上把 Irisy 端到端跑通 (Pi 检测/启动 → 热键唤起 →
chat 收发 → 记忆写入),逐条修掉 mac-only 假设。本 handoff 跟踪整个 Windows
bring-up。

## 证据

P0 brain 破绑 (已修):
- `pi-detect.ts:whichBin` 用 `command -v` + `shell:'/bin/sh'` → Windows 无 `/bin/sh`,
  PATH/npx 检测全废 → `detectPi()` 在 Windows 直接抛 `PiNotFoundError`,Irisy chat 死。
- `~/.local/bin/pi` 是 unix-only 候选,Windows 全局 npm 在 `%APPDATA%\npm\pi.cmd`。
- `spawnSync` 直跑 `.cmd` shim 在 Windows 需 `shell:true`,且 `where` 会返回无扩展名的
  unix shim (不可执行) 需优先选 `.cmd`/`.exe`。

待查 (下一步):
- `pi-bridge.ts` RpcClient `cliPath: pi.command` → Pi upstream 在 Windows 能否 spawn `.cmd`
  (`pi --mode rpc`)。Pi 0.73.1 是跨平台发布,大概率已处理,需真跑确认。
- 4 个 cfg-gated shell 文件 (hotkey/window/updater/lifecycle) 的 Windows 分支完整度。
- `pi_install.rs` 懒安装在 Windows 的路径。

## 建议

逐链修 + 真跑验证: detect (done) → spawn → 热键/窗口 → chat E2E → 记忆。
每条独立 commit,带 [H-2026-06-08-002]。
