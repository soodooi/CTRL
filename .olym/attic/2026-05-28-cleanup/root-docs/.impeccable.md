## Users

CTRL is a keyboard-driven AI cockpit. Primary users:

- **bao** — solo founder / power user, builds the product. Lives in dark
  rooms at odd hours; eyes need calm contrast, not glow.
- **Creators** — future keycap authors who use CTRL's workshop (`/forge`)
  to build keycaps. Need dense IDE-style information.
- **OPC adopters** — globally English-first per `decision_ctrl_is_global_english_first`.
  Chinese is i18n adapter, not the design baseline.

**Usage context**: persistent floating window (920×560 base), summoned by
the Ctrl key, lives on top of whatever else the user is doing. The
cockpit is **always there**; it has to not annoy.

## Brand Personality

Three words: **industrial · instrumental · unimpressed**

- **Industrial** — cockpit / aviation PFD vocabulary, OP-1 Teenage
  Engineering, Linear's calm restraint, Braun. Not "tech-y."
- **Instrumental** — every element earns its place. No decoration without
  purpose. Motion that explains state, not motion as garnish.
- **Unimpressed** — doesn't try to be flashy. Calm confidence. Won't
  shout for attention. The Raycast standard: "search + act," not
  "search + dazzle."

**Anti**: ChatGPT gray slabs · Material Design templates · purple-blue
SaaS gradients · "AI sidebar" hovering chat blobs · glassmorphism ·
neon-on-dark · bouncy elastic motion · glow halos that don't carry state.

## Aesthetic Direction

**Theme**: **light** by default (bao 2026-05-25: "试试明亮色"). Dark
is opt-in via `data-theme="dark"`, not system-followed. The pivot keeps
the industrial / instrumental / unimpressed personality — brightening
the surface doesn't mean adding glow, gradients, or sparkle.

Anti-references still hold under light: no purple-blue SaaS gradients,
no glassmorphism, no FAB / ripple, no decorative halos. The cockpit is
still calm; it's just on paper now.

**Inspirations** (≠ copy targets):
- **Linear 2026-03 refresh** — dimmer sidebar, warmer gray neutrals,
  scaled-down icons, softened dividers, "calmer interface for a product
  in motion"
- **OP-1 Teenage Engineering** — industrial typography, monospace
  readouts, every label earns its place, color = signal
- **Raycast** — keyboard-first, minimal palette, search + act
- **Cursor IDE** — calibrated contrast, dense info layout, accent =
  identity not decoration
- **Frank Chimero** — restraint, intentional weirdness, content over chrome

**Anti-references** (memorize, will not ship):
- ChatGPT 灰
- Material Design ripples / FAB / etc.
- Purple-blue SaaS template gradients
- Sidebar that competes with content for attention
- Halos / glows / decorative motion loops that don't carry state

## Design Principles

1. **Hands stay on keyboard** — Raycast principle. Every primary action
   has a keyboard shortcut. Cmd+K is industry standard (Linear / Figma /
   Notion / Vercel / Raycast / Cursor 2026), CTRL inherits.

2. **Content > chrome** — Linear 2026 dimming applied: the keycap
   workspace area is the star. Sidebars (left keyboard + right rail)
   are dimmer than the main workspace surface, not brighter.

3. **Motion explains state, never decorates** — 2026 trend confirmed by
   uxpilot / envato research. No halos for halos' sake. Mascot
   breathes (idle signal) — that earns its motion. StatusBar pulse on
   LIVE (signal) — earns it. Ambient mesh gradient on workspace — cut.

4. **Density via mono labels + tabular numerals** — cockpit instruments
   tell stories at a glance. KRN ● MESH ○ LLM ● reads in 0.3s.
   Minimum mono font size: **10px** (cockpit floor). 9px retired —
   bao 2026-05-24 explicit veto ("字看不清楚").

5. **One signature accent** — CTRL blue (`--color-accent`,
   `var(--ctrl-blue-light)`). No secondary accents in chrome. Other
   colors (warmth, status, keycap) appear only in their semantic
   contexts. Per Linear's accent-as-identity 2026 pattern.

6. **Calm familiar over novel gestures** — no swipe / pinch / shake
   unless industry-standard (drag-reorder = expected; scroll-driven
   reveal = trendy but optional). Users are tired of figuring out
   gestures (envato 2026).

7. **Filesystem-as-truth** — per `decision_ctrl_obsidian_philosophy`:
   the UX layer is augmentation, vim test passes. Don't design UIs
   that imply data lock-in. Markdown + YAML behind every surface.

8. **No AI slop tells** — banned patterns (will be grep-able):
   - Side-stripe borders > 1px on cards / list items / callouts
   - Gradient text (`background-clip: text` + gradient bg)
   - Bento-icon-rounded-rect templates above every heading
   - Glassmorphism (blur on glass cards)
   - Drop-shadow rounded-card layouts
   - Hero "big number + small label + supporting stats" templates

## Token-Level Direction (v0.3)

**Neutrals**: shift from cool-blue gray (H=264°) → warm gray (H=80° low
chroma 0.005). Linear's 2026 direction validates this; OP-1 industrial
warmth supports it.

**Type scale** (fixed `rem` for app UI per impeccable guidance — no fluid
clamp in product chrome):
```
mono-xs   10px  cockpit minimum (KRN / MESH labels)
mono-sm   11px  primary mono (tape readouts, version)
mono-md   12px  emphasized mono (Settings tab labels)
xs        12px  small body / hint
sm        13px  body
base      15px  primary body
lg        18px  section heading
xl        24px  page title
2xl       32px  hero
```
NO 9px. NO clamp() in product UI. Marketing surfaces unchanged.

**Active state vocabulary**: softer. `border-color 60% accent` +
`bg-color 6% accent tint`, no glow / ring shadow. 100% accent reserved
for focus ring + primary CTA only.

**Motion**:
- chrome (active swap / focus): 150ms `--ease-out-quart`
- micro-feedback (button settle): 220ms with subtle scale (NOT bounce)
- decorative ambient: only when carrying state (Irisy breath = mood
  signal). Everything else cut.
