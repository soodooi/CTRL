# 0002 — 北极星：反 Trae，人人用得起用得会

> bao 2026-06-12 亲述定位。这比任何功能都重要，是 CTRL 的灵魂。
> 锚点：[[0001-why-users-love-ctrl]] · [[0003-ctrl-is-modular-intent-platform]] · [[0006-converged-architecture]]

## 一句话

让大家用 AI **更简单、更傻瓜、不需要弯弯绕**——人人用得起、用得会。

bao 的原话动机：用 Trae 太复杂、搞不清；想做一个**大家都能用**的工具。

## Trae 的病 = CTRL 的反面

Trae（及多数 AI IDE / Coze / n8n）的复杂在于：**一堆面板、一堆概念、要你先学会它，你才用得了。** 信息一次性砸脸。

CTRL 反过来：**用户什么都不用学，对着一个框说人话，AI 替他把弯弯绕全咽掉。**

> CTRL 真正的护城河不是功能多，是「别人要你学，我们替你想」。

## 傻瓜测试（新设计门槛，与 vim test 并列）

> 一个从没用过 AI、不懂「模型 / 配置 / 功能包」的普通人，第一眼看这个界面，**不用人教，知道点哪、能拿到东西吗？** 答 No = 太绕，重做。

每加一个东西先过傻瓜测试；过不了就**藏进 Irisy 后面**，别摆到用户面前。

## 对当前产品的校准（2026-06-12）

**对的（守住、放大）**
- **Irisy 是唯一入口** = 最大的反-Trae 武器。Trae 让你管理工具，CTRL 让你只管说话。
- 高频钩子一键填话术、默认零配置能跑、空态给人话引导。

**还「绕」、违背傻瓜测试的（待改）**
1. **「功能包」这个词**普通人不懂。理想：Irisy 听懂需求**自动装好**，用户不用知道这个概念、不用去 Discover 挑装配。
2. **打开 Notes 时四区同现（L1+Irisy+L2+输出）**——像 Trae 的「信息砸脸」。普通人会一愣。
3. **填 API key（BYOK）**劝退普通人。默认必须**开箱即用、不填任何东西就能聊**（CF Workers AI 默认路径，BYOK 永远后置）。

## 推论

- 概念越少越好：能不让用户知道的（功能包 / 模型 / 配置 / mcp），就藏进 Irisy。
- 默认即用：第一次打开，不配置就能拿到价值。
- 一个入口：所有能力从 Irisy 一句话进，不让用户在面板间导航。
