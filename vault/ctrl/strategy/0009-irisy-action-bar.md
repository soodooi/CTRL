# 0009 — 输入框上方「功能包栏」要履行哪些业务

> bao 2026-06-13:「这样列举的方式不对,你要知道这个栏有哪些业务要履行。」
> 锚点:[[0007-irisy-ambient-context]] · [[0008-ui-texture-pass]] · `.kiro/steering/development-philosophy.md` § one-shot / § AI is a pipe / § transparency · `.kiro/steering/mcp-development.md` · memory [[project-ctrl-modular-intent-platform]]

## 这栏是什么(定位)

**Irisy 输入框正上方那一行 = 用户按回车前的「装填台 + 功能包能力橱窗」。**
它回答用户三个问题:**① 这次对什么操作?② 用哪个能力?③ 还能装什么能力?**——并由 Irisy 用工作区上下文(0007)主动把最相关的推到最前。

## 现状错在哪

当前 `QUICK_HOOKS` = 5 个写死 chip(Translate/Polish/Summarize/Actions/Plan),平铺列举。错在:
- **静态**:不管你装了哪些功能包(mcp),永远这 5 个 → 跟 mcp 生态脱节(`.kiro/steering/mcp-development.md`:mcp = 用户装的原子能力)。
- **无 ambient**:不管你现在选中了什么、打开了什么(0007 Irisy 读工作区),永远一样 → 浪费了 Irisy 的 context 感知。
- **无对象**:不能指定「对什么处理」(选区/剪贴板/截图/当前文件) → 违背「AI 是 pipe,in-line 处理」(设计哲学 #5)。
- **无扩展**:点不出「装更多 / 让 Irisy 造一个」→ 这栏本该是功能包生态入口,却是死路。

列举 ≠ 规划。先定业务职责。

## 这栏必须履行的 5 项业务

1. **对象装填(对什么)** — 声明这次让 Irisy 处理的素材。两个来源:
   - *ambient 自动*:工作区当前焦点(打开的笔记/网页/选中的文字),Irisy 已读到(0007),默认带入。
   - *主动附件*:选区 / 剪贴板 / 截图 / 文件,用户点着加。
   - *隐私边界*:标记隐私的(日记/私密浏览)**不带入**(0007 隐私例外),且这栏要让「带了什么/没带什么」可见(transparency by drill-down,设计哲学 #6)。

2. **能力触发(用什么)** — 已安装功能包(mcp)暴露的 **one-shot 原子动作**(一个 mcp = 一个动作,无 wizard,设计哲学 #4)。**有哪些 = 你装了哪些功能包决定**,不是写死。

3. **ambient 排序(谁排前面)** — Irisy 读工作区 → 把当前最相关的动作推到最前:选中中文→翻译;打开 PDF→摘要/提取;看代码→解释/重构。**scale 在 registry,UI 只露 top-N + 「更多」**(intent platform:让 UI 更简单,长尾在注册表)。

4. **扩展入口(还能装什么)** — 没有想要的能力时:去 Discover 装功能包 / 让 Irisy 用自然语言**现造一个 mcp**(`vault/ctrl/specs/connectors/spec.md`:AI 创作助手生成 manifest)。这栏是生态入口,不是封闭列表。

5. **边界透明(状态)** — 当前用哪个 model(已落第一行,不重复)+ 隐私边界可见。

## 信息架构(一行内的分区,承载上述业务)

```
┌───────────────────────────────────────────────────────────────┐
│ [对象] ⧉选区 ·📋剪贴板 ·🖼截图   [功能包动作 top-N, ambient 排序]  ⊕ │
│  └业务1: 对什么            └业务2+3: 用什么能力, Irisy 排序   └业务4│
├───────────────────────────────────────────────────────────────┤
│ [ Ask Irisy, or pick an action above… ]                      ↑ │
└───────────────────────────────────────────────────────────────┘
```

- **左段 = 对象装填**:当前带入的素材(ambient 焦点 + 主动附件),可点掉/可加。空闲时收起。
- **中段 = 功能包动作**:已装 mcp 的 one-shot,Irisy context 排序,只露前几个。
- **右段 ⊕ = 更多/装**:展开全部已装动作 + 「装功能包」+「让 Irisy 造一个」。

## 从用户视角(bao 2026-06-13:「业务差不多,从用户角度考虑」)

上面 5 项是**系统功能**视角。用户视角差别很大:**用户是带着「具体的当下」来的,不是来逛功能列表。** 用户脑子里不是"我要调用翻译功能",而是「**这个东西 + 这件事**」(刚看到的英文 → 它说啥;写的邮件 → 说得体点;截的报错 → 咋解决)。「列举固定 chip」错在假设用户每次面对同一张空白选择题,无视了用户每次都带着不同的当下。

用户视角改变 4 个设计决定:
1. **认出用户带的东西**(不用他"声明对象"):复制/选中/截图,栏主动接住。
2. **说人话,不露「功能包/mcp」**:用户看到"翻成中文",不是"调用翻译 mcp"。
3. **少选择**:对带的东西直接摆好最可能的 1–3 个意图,确认即可,不是从一堆里挑。
4. **个人化**,不是产品钦定 5 个;新用户给引导性默认。

## ui 极简(bao 2026-06-13:「ui极简」—— CTRL 根哲学,anti-ref:Material / SaaS 工具栏)

把业务 + 用户视角收敛成最极简形态。**输入框是主角,这栏是按需浮现的一行纯文字:**

```
默认(没素材):     只有输入框, 栏隐身
有素材/context:   翻成中文  润色  说重点                    更多
                  └纯文字, 无图标/无边框/无背景, ≤3         └一个入口收长尾
```

4 个极简决定(对比原 5 个带 emoji 带边框的常驻 chip):**① 默认隐身**(没东西可建议就不占位)· **② 纯文字**(去 emoji/边框/背景,Raycast/Linear 不是工具栏)· **③ ≤3 个**(少选择)· **④ 一个「更多」收长尾**。chip 只是给懒得打字的捷径,用户始终能直接跟 Irisy 说。

**一句话用户价值:「我手上的东西 + 我想做的事,不用想怎么用 Irisy,顺手一点就成。」**

## 落地次序(不破坏现状)

1. ✅ **业务职责 + 用户视角 + 极简形态**已锁(本文)。
2. ✅ **极简 UI 骨架已搭**(0.1.273):5 个 emoji chip → 3 个纯文字 + 「More」,无边框无背景,输入框是主角。`AmbientHome.tsx` `QUICK_HOOKS`。
3. ⬜ **逻辑(后续,跟 [[0007-irisy-ambient-context]] 一起)**:ambient 读工作区 → 认素材 + 排序 + 个人化;「默认隐身 / 随状态变形」;对象装填(选区/剪贴板/截图);隐私过滤;「让 Irisy 现造一个 mcp」。
