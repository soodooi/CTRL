## 1. 产品定位

官方话术 **「像表格一样简单,像数据库一样强大」**。本质 = **多视图关系型数据库 + 协作 + 自动化 + AI 的轻量低代码平台**(多篇第三方直接称其"轻量版低代码平台"、"后端管理平台替代")。

## 2. 字段类型 (强类型是底座)

- **基础**:文本、多行文本、数字、单选、多选、日期、复选框、评分、进度、货币、条码、电话、邮箱、超链接、地理位置、附件
- **元数据**:人员/群组(连通讯录)、创建时间、修改时间、创建人、修改人、自动编号
- **关系型**(见 §3)、**公式**、**按钮**、**AI 字段/字段捷径**(见 §5)
- 视觉特点:单/多选 = 彩色标签胶囊,评分 = 星,进度 = 进度条,关联 = 可跳转引用

## 3. 关系型字段 (飞书的灵魂,实证细节)

- **单向关联**:A 表记录指向 B 表(产品明细 → 产品)
- **双向关联**:两张表都能看到彼此关联记录(合同 ↔ 产品明细、合同 ↔ 回款记录)
- **查找引用 (Lookup,VLOOKUP 式)**:沿关联 + 条件,把另一张表的字段值自动取过来(按产品 ID 匹配取原价)
- **汇总 (Rollup)**:对关联表的多条记录聚合(对同一合同所有产品小计求和)
- **公式**:基础(单价 × 数量)+ 条件聚合(`FILTER` 逐条判断 + `SUM` 条件求和)
- **多表协同**:关联 + 公式自动更新 →「选合同后自动显示总额和剩余回款金额」

## 4. 视图 (一份数据多视图)

官方 6 种:**表格 Grid / 看板 Kanban (按字段分组、拖拽换列=改值) / 画廊 Gallery (卡片墙) / 甘特 Gantt (按起止日期画彩色时间条,可按优先级/状态着色) / 日历 Calendar / 表单 Form (对外收集,提交即新增一条记录)**。

核心机制:筛选 / 排序 / 分组(支持**多级分组**)/ 隐藏 / 冻结 / 行高 是**视图自身属性**,不动底层数据;一表多视图各看各的。

## 5. AI 能力 = 字段捷径 (★ 差异化核心,实证细节)

**字段捷径 (Field Shortcuts)** = 把 AI / 公式 / API 能力**封装进字段**,一次配置 → 整列批处理。官方提法「会用表格就会用 AI」。

- **AI 能做**:分类、智能打标、信息提取、AI 搜索、总结、翻译、自定义 AI 自动填充
- **配置流程实证**(以 DeepSeek R1 字段为例):列首 `+` → 搜模型 → 选「DeepSeek R1」→ 关联账号(100 万 token 免费额度)→ 填**输入指令**(提示词)→ 选**引用字段**(要分析的列)→ 输出「思考过程」+「输出结果」两字段
- **智能标签字段**:数据源选某列 → 预定义标签选项 → 自定义匹配规则 → 勾「自动更新」整列跑
- **信息提取字段**:选源列 → 指定提取项 → 自动更新批处理
- **整列批处理机制**:所有 AI 字段都有「自动更新」选项,新增行自动跑 AI,无需逐行手动触发
- **模型生态**:直连 DeepSeek;**字段捷径市场**有第三方大模型入驻(智谱 / 月之暗面 / 零一万物)+ API 请求工具
- **AI 智能分析仪表盘**:一键获取仪表盘数据背后的问题与变化;支持 **AI 一键生成**仪表盘 / 自动化群推送

## 6. 自动化 (实证)

可视化「**触发器 + 条件 + 动作**」,零代码。

- **触发器**:定时 / 点击按钮 / 记录变更(新增·修改)/ 表单提交
- **动作**:发送通知(群消息卡片审核)/ 更新字段 / 创建数据 / 发邮件 / 调 webhook
- **工作流 = 自动化升级版**(更丰富节点、更复杂任务)
- **vs n8n**(第三方评测):优势 = 可视化看数据流转、可对多行**并行执行**、上手门槛低

## 7. 仪表盘 (实证)

基于表数据搭图表,自定义网格布局。**8 种可视化**:柱状图、折线图、饼图、条形图、散点图、组合图、漏斗图、统计数字。支持 **AI 一键生成**仪表盘、定时 / 条件发送。本质 = 只读派生视图。

## 8. 数据操作 / 协作 / 集成

筛选(多条件)、排序(多列)、多级分组、隐藏字段、冻结列、行高、全表搜索、记录详情卡、实时协同、评论 @、字段级编辑历史、四级权限(表 / 视图 / 字段 / 记录;管理员可改全部内容 + 权限 + 自动化 + 仪表盘)、导入导出 Excel·CSV、Open API / Webhook、模板中心、共享数据源。

## 对 CTRL 智能表格的提炼 (决策详见 ADR-003 §6 v10)

| 飞书特点 | CTRL 对齐 |
|---|---|
| 字段即强类型 + 渲染即类型 | 7 类型扩彩签/星级/进度 ✅ (§6.2) |
| 一份数据多视图(视图 ≠ 数据) | `.md` 派生 表格+看板,视图态存 frontmatter `view.*` ✅ (§6.2) |
| 记录详情卡 | 行展开成卡 ✅ (§6.2) |
| 关系型(关联/Lookup/Rollup) | `[[wikilink]]`+backlink **软关联**替代(最大取舍) (§6.4) |
| **字段捷径 = AI 封装进列 + 整列批处理** | **差异化主战场**:AI 字段经 Irisy/gate,「自动更新」整列批处理直接借鉴 ✅★ (§6.3,呼应 §8.2-F AI-as-column) |
| 自动化流程编辑器 | 撞「不做清单」**永久排除** ❌ (§6.4) |
| 仪表盘 8 图表 + AI 一键生成 | 只读派生 + AI 生成,v1.x (§6.4) |

**最大启示**:飞书把 AI 做成「**字段捷径**」——AI 不是旁边的 chatbot,是**长在列上、整列自动跑**的能力。这跟 CTRL「AI 是 pipe 不是 sidebar」哲学高度一致,也正是 ADR-003 §8.2-F 早已锁定的「AI-as-column」。智能表格是这个理念在表格面的落地形态。
